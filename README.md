### オリジナル探索ゲーム(MISORIA:Frontier（仮）)
TypeScript + React を用いて制作した，
探索・ストーリー演出を組み込んだオリジナルゲームです

本作は当初マインスイーパーをベースとして開発を開始しましたが，現在はその要素を簡略化し，
マス移動型の探索ゲームとして再設計を進めています．

### 制作目的
- モダンな Web 技術（TypeScript / React）を用いた
  アプリケーション制作経験の習得
- UI・状態管理・ゲームロジックを分離した拡張性の高い設計の修得
- 自身のオリジナル世界観をゲームという形で表現・発信すること

### ゲーム概要
- プレイヤーはマス上のマップ上を移動しながら探索を行う
- マスには以下の要素が配置される
  - 危険マス
  - アイテムマス
  - ストーリーイベントマス
  - ゴール
- キーとなるアイテムを回収後，ゴールマスに到達するとクリア
- ストーリー画面やアイテムログを通じて世界観が進行

マインスイーパーの「マス構造」を活かしつつ，探索・進行を重視したゲーム体験を目指しています．

### 主な機能
- ゲームシステム
  - マス単位での移動処理
  - マス到達時のイベント判定
  - アイテム回収・管理
  - ゴール到達によるクリア判定
- ストーリー演出システム
  - 探索中のログ表示
  - チャプター開始時・クリア時のストーリー画面
- 画面遷移・シーン管理
  - タイトル画面
  - チャプター選択画面
  - ストーリー画面
  - ゲーム画面
  - クリア後遷移処理

### 使用技術
- **TypeScript**
  - 静的型付けによる安全性の高い実装を行い、  
    バグの抑制や将来的な機能拡張を見据えて採用

- **React**
  - コンポーネント指向による画面構成を採用
  - シーン管理・状態管理を明確に分離して実装


### 工夫した点
- 画面単位で `components` を分割
  - Title / ChapterSelect / Game / StoryPanel /StoryScreenなど
- ゲームロジックを `logic` ディレクトリとして分離
  - マス生成，再帰開放、勝利判定などを UI から独立させて実装
- これにより
  - UI とロジックの責務分離
  - 修正やデバッグの容易化
  - 章追加やシステム追加への高い拡張性

を意識しました
